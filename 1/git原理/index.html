原理数据基础：

  • Blob对象： 存储文件的内容，和文件名无关。
  • Tree对象： 存储文件名（或目录名）、文件（目录）对应Blob（Tree）名、文件（目录）权限，数据类型
  • Commit对象：当前Tree名，父Commit名称，附加信息（作者、提交时间、注释等）

原理实现：

  • git hash-object: 创建Blob对象，存储文件内容
  • git update-index: 追加的方式更新暂存区（.git/index）
  • git write-tree: 基于暂存区创建Tree对象
  • git commit-tree: 创建Commit对象

以上命令中，git add 就是1、2、3的封装，git commit就是4的封装。

Tips:

  • 当前git仓库文件状态，就是当前HEAD所指定Commit对象中tree对象以及起连接的blob对象的集合。
  • refspec: 只要存在对象数据的流动，就涉及到refspec，其就是:结构，告诉流动的源和目的， 例如：refs/heads/master:refs/remotes/origin/master；refs/tags/V1:refs/remotes/origin/tags/v2
  • fetch: 下载远程对象数据（本地没有的）到.git/objects，并修改相应远程分支的指向的 commit更新到下载的最新commit。
  • merge: 通过（递归）三路合并算法按提交顺序，将commit对象合并进指定分支。
  • rebase: 当前Commit对象A，rebase的commit对象B, A和B的共同祖先commit C，将C到A之间所有commit 复制后，将当前分支指向B，并依次应用所复制的commit。实际上rebase可以指定想要复制的起始commit。 当多人同时操作一个仓库时，谨慎使用rebase。
  • reset: 移动HEAD和分支所指向的祖先commit；用HEAD所指commit更新暂存区（index）；更新工作去数据。
  • revert: 提交一个新的反向提交去撤销之前的某一个提交。
